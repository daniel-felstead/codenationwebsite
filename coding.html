Beginning this week I didn't know how to code whatsoever, I was familiar with some of the concepts and terms and had a very rudimentary knowledge of HTML from peeking at source code of websites. I even owned books on HTML, CSS and jQuery, but I was never encouraged to work quietly in a room and learn alone in this way.

I saw Code Nation advertised and I knew immediately this was something that I wanted to pursue, the interest was there - just not the learning method.

Learning with people and having demonstrations of the basic concepts really helps me understand what can be done and what possibilities exist.

When we started the course we were presented with the code for i in range(10): which of course was completely foreign.

For the week I learned Python my knowledge expanded to a point where I was confident enough to write code, think of solutions and pursue ideas with a try and reassess if it didn't work attitude.

I have had so much fun learning it, particularly the free-reign second week where we were tasked to build a game.

Not only was the learning experience fantastic, I've learned key insights and a passion for making things with code, but also probably most importantly the people. I really believe that bouncing ideas and code snippets around has had a huge impact and increased the overall enjoyment of the course and coding.

I would love to pursue a career in it, I think working collaboratively on code in this manner is excellent and I would encourage anyone who wants to learn code: try!

HTML & CSS

.article { }

At a glance HTML & CSS seemed simple when compared to Python. This was my error. Although the languages are easier to understand, working with them in practice is not as straightforward as you have a visual representation to work with and tweak. Small changes can become complex problem solving practices.

We learned the basics of these languages at the beginning of the third week. By the end of it we needed a functional website (which you are now viewing), a daunting task indeed. One that required extra research and a lot of trial and error.

I had originally written the CSS with a thought processes that was too logical, I ended up reworking it and starting the CSS from scratch with a structure in my head of how to lay it out. I am happier with the result, but I also know that with practice I can get better at this.

Overall I have enjoyed learning HTML and CSS, as with Python I would recommend at least trying to learn the basics. What's the worst that can happen?

I am truly grateful for the three weeks of learning that I have had, particularly of Keira @ Code Nation who has made this a phenomenal experience.
